Project Settings

Control how search behaves in this project.
Enable Background Precompute
Preloads semantic theme-cluster seeds when you open the project so the cluster modal can start faster later. Recommended ON for higher-performance devices, OFF when conserving battery.
Use LLM Theme Naming
Uses your configured LLM API to rename Theme Cluster labels into clearer wording. This only updates labels and does not change clustering assignments.
Preload Zotero Sync
Background-load Zotero folders and unmatched-paper previews when you open this project, so the Zotero Sync panel opens instantly later.
Use citation-count quality assist
When enabled, similarity blends text relevance with a compressed citation-count score from OpenAlex. This only adjusts ranking similarity, not the core text embeddings.
Citation Weight Strength
Citation count is compressed with min(log(1 + citations) / log(1 + 200), 1) before blending into similarity.
Similarity Scores Need Refresh
Keywords, abstract, or current content changed. Re-run similarity scoring to refresh the project ranking.

StarMap System Introduction

A fast-start guide for building a literature reference library around your own target thesis.
Quick Start Guide
Build a Thesis-Centered Reference Library Fast
StarMap works best when you use each project as a focused research workspace, not a giant paper dump. The fastest path is to define the thesis clearly, import a strong seed set, clean the ranked list, then expand outward through citations and Zotero.
1 project = 1 thesis direction
Start small, then expand
Top Related Papers = working queue
1
Define the Thesis Clearly
Create a project with a clear title, a working abstract, and any draft content you already have. StarMap ranks papers against this specific thesis direction, not against a vague topic label.
2
Import a Strong Seed Set
Begin with a small batch of papers that are already close to your topic. A focused first import gives StarMap a much cleaner signal than dumping hundreds of weakly related papers at once.
3
Work From Top Related Papers
Use Top Related Papers as your main review queue. Search, filter, and sort the list, then label papers as Core, Pending, or Underweight to shape the reference library around your argument.
4
Expand Through Nearby Papers
After the seed set is stable, use Cited By, References, and Zotero Sync to pull in papers from the same literature neighborhood. This is the fastest way to grow the library without losing focus.
5
Read, Annotate, and Detect Themes
As you read, add notes and update paper status. Once the project has enough depth, use the map views and Auto Cluster Themes to detect sub-themes and see where your library is dense or still missing coverage.
6
Keep Watching the Frontier
Once your core library is stable, use Project Literature Watch to scan for newly published papers that still fit your thesis direction or the scholars you care about. Review recommendations first, then add only the ones that truly deserve a place in StarMap.
7
Turn the Library Into Writing
When the reference library stabilizes, use Paper Status Overview to select the papers that should feed a literature review draft. Export BibTeX when needed, but keep using StarMap as the place where your thesis, evidence base, and notes stay connected.
Rule of Thumb
If you want the fastest path to a useful literature library, follow this sequence: define the thesis clearly → import a strong seed set → clean the ranked list → label important papers → expand outward through citations and Zotero → use Project Literature Watch to keep the library current. That usually gets you to a focused reference base much faster than importing everything first and organizing later.
